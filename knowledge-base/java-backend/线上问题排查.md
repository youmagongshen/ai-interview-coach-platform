# 线上问题排查

## 接口响应慢排查步骤

### 1. 确认问题范围
- 是单个接口还是所有接口都慢？
- 是部分用户还是所有用户？
- 是偶发还是持续？

### 2. 检查服务器资源
```bash
# CPU使用情况
top
# 磁盘IO
iostat -x 1
# 内存使用
free -h
# 网络状况
netstat -an | grep ESTABLISHED
```

### 3. 检查应用状态
```bash
# Java进程
jps -l
# 线程堆栈
jstack <pid>
# 内存 dump
jmap -heap <pid>
```

### 4. 检查数据库
- 慢查询日志是否开启
- 是否有慢查询（执行时间>1s）
- 数据库连接池是否耗尽
- 是否有锁等待

### 5. 检查缓存
- Redis是否正常
- 命中率如何
- 是否存在大key

### 6. 检查日志
- 应用错误日志
- GC日志
- 业务日志

### 7. 链路追踪
- 检查调用链耗时
- 定位具体哪个环节慢

## 常见原因

1. **数据库**：慢查询、缺索引、连接池耗尽
2. **CPU**：计算量大、死循环、频繁GC
3. **内存**：内存泄漏、OOM
4. **IO**：磁盘IO瓶颈、网络延迟
5. **缓存**：缓存击穿、穿透、雪崩
6. **第三方**：外部接口超时

## 快速止血方案

1. 扩容（增加实例）
2. 限流（熔断降级）
3. 降级（非核心功能）
4. 回滚（如果是新上线导致）
