# 数据库死锁

## 什么是死锁

两个或多个事务相互等待对方释放锁，导致都无法执行。

## 死锁产生条件

1. 互斥：资源只能被一个事务占用
2. 占有并等待：事务占有资源同时等待其他资源
3. 不可抢占：不能强制抢占其他事务的资源
4. 循环等待：形成事务等待环路

## 死锁示例

```sql
-- 事务A
UPDATE account SET balance = balance - 100 WHERE id = 1;  -- 锁住id=1
UPDATE account SET balance = balance + 100 WHERE id = 2;  -- 等待id=2

-- 事务B（同时执行）
UPDATE account SET balance = balance + 100 WHERE id = 2;  -- 锁住id=2
UPDATE account SET balance = balance - 100 WHERE id = 1;  -- 等待id=1（死锁！）
```

## 解决死锁

### 1. 调整SQL顺序
统一所有事务访问表的顺序。

### 2. 降低隔离级别
使用 READ COMMITTED 代替 REPEATABLE READ。

### 3. 减小事务范围
- 减少事务内SQL数量
- 及时提交事务
- 避免在事务中交互

### 4. 索引优化
有索引能减少锁的范围。

### 5. 定时检测
```sql
-- MySQL查看死锁
SHOW ENGINE INNODB STATUS;
```

## 预防死锁

1. 固定的资源访问顺序
2. 小事务短事务
3. 合理使用索引
4. 加锁超时机制
