# 项目介绍模板

## 项目基本信息

1. **项目名称**：XXX系统
2. **项目类型**：toB / toC / toG
3. **项目规模**：多少开发人员、历时多久
4. **技术栈**：后端语言、框架、数据库、缓存、消息队列等

## 你负责的模块

- 核心业务模块设计
- 核心接口开发
- 性能优化
- 技术难点攻关

## 项目亮点（STAR法则）

### Situation（背景）
业务快速发展，原有系统无法支撑。

### Task（任务）
需要重构/开发新系统。

### Action（行动）
- 设计了分布式架构
- 引入缓存减少数据库压力
- 实现了读写分离

### Result（结果）
- QPS从100提升到5000
- 响应时间从2s降低到200ms

## 常见难点及解决

1. **高并发**：Redis缓存 + 限流 + 异步
2. **大数据量**：分库分表 + 读写分离
3. **分布式事务**：TCC / 最终一致性
4. **接口性能**：索引优化 + SQL优化 + 缓存
5. **数据一致性**：分布式锁 / 消息队列

3. 了解核心技术选型原因
4. 能说出具体的优化数据
5. 诚实回答，不会的不要硬撑
