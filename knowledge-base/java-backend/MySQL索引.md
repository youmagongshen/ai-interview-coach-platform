# MySQL索引

## 什么是索引

索引是一种数据结构，能快速定位数据，类似于书的目录。

## 索引的优缺点

### 优点
- 加快查询速度
- 加速排序
- 唯一索引保证数据唯一性

### 缺点
- 占用磁盘空间
- 降低写入速度（INSERT/UPDATE/DELETE需要维护索引）
- 索引并非越多越好

## 索引类型

1. **主键索引**：唯一且不为空
2. **唯一索引**：值唯一，可为空
3. **普通索引**：普通字段索引
4. **全文索引**：用于文本搜索
5. **复合索引**：多列组合索引

## 索引数据结构

### B+树（InnoDB默认）
- 非叶子节点只存储键值，不存储数据
- 叶子节点包含所有数据并用链表连接
- 适合范围查询

## 不适合建索引的情况

1. 数据量小的表
2. 频繁更新的字段
3. 区分度低的字段（如性别、状态）
4. 很少查询的字段

## 索引优化技巧

1. 遵循最左前缀原则
2. 避免在索引列上使用函数
3. 使用覆盖索引减少回表
4. 避免全表扫描
5. 定期分析和优化表
