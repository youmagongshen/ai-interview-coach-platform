# Redis缓存

## 缓存穿透

### 概念
查询一个不存在的数据，数据库也没有，每次请求都会穿透到数据库。

### 解决方案
1. **布隆过滤器**：将所有存在的数据存入布隆过滤器，请求来时先查布隆过滤器
2. **空值缓存**：即使查询结果为空，也缓存一个空值，设置较短过期时间
3. **接口校验**：做好参数校验，拦截非法请求

## 缓存击穿

### 概念
某个热点key过期瞬间，大量并发请求同时访问数据库，导致数据库压力剧增。

### 解决方案
1. **互斥锁**：用Redis的SETNX保证只有一个请求去查数据库
2. **永不过期**：不设置过期时间，由后台异步更新
3. **逻辑过期**：存储时添加逻辑过期时间字段，过期后返回旧数据同时异步更新

## 缓存雪崩

### 概念
大量缓存数据在同一时间过期，或Redis故障，导致大量请求同时访问数据库。

### 解决方案
1. **随机过期时间**：给缓存过期时间加随机值
2. **Redis集群**：主从复制 + 哨兵/集群
3. **多级缓存**：本地缓存 + Redis缓存
4. **熔断降级**：接口加熔断器
5. **限流**：使用Redis漏桶算法限流

## Redis数据类型

- String：字符串
- Hash：哈希表
- List：列表
- Set：无序集合
- Sorted Set：有序集合
- Bitmaps：位图
- HyperLogLog：基数统计
- Geospatial：地理位置

## Redis持久化机制

1. **RDB**：定时生成数据快照
2. **AOF**：记录所有写命令

## 缓存使用策略

1. 读：先查缓存，缓存没有查数据库
2. 写：先更新数据库，再删除缓存（而非更新缓存）
